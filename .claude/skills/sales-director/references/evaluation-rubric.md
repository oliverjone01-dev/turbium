# Методика оценки менеджера отдела продаж GENGROUP

**Owner:** sales-director skill
**Last calibration:** 2026-06-30
**Next review:** 2026-09-30 (Q3 close)
**Source:** Костины анализы (10 менеджеров) + sales-director опыт + Bitrix24 C49 поля

## 9 параметров + веса + cut-offs

### 1. Конверсия close-won (CR2) - вес 25%

**Формула:** `WON / (WON + LOSE + APOLOGY)` за rolling 90 дней, по cohort (B2B / B2C / Дилер отдельно).

**Цели по сегментам** [ГИПОТЕЗА: на основе HubSpot industry benchmarks B2B Pro 2023 + Salesloft pipeline reports; не валидировано на GENGROUP cohort. Требует Q3-2026 калибровки на реальных данных rop.json после 90 дней stable measurement]:

| Сегмент | Target | OK | Внимание | PIP-zone |
|---|---|---|---|---|
| B2B custom-furniture | 18-22% | ≥16% | 10-15% | <10% |
| B2C designer-furniture | 8-12% | ≥7% | 4-6% | <4% |
| Дилер reorder | 25-35% | ≥22% | 15-21% | <15% |

**Sanity check:** диапазон 25-35% для дилеров ≈2x ширины (P9 hard rule «диапазон шире 2x = пометить»). Это [ШИРОКИЙ ДИАПАЗОН - НЕПРОВЕРЕНО]. Требует уточнения по cohort (новый дилер vs устоявшийся).

**Anti-cheat:** считать только сделки с `budget > 0`. Сделки budget=null исключаются (это либо junk-лиды, либо менеджер не проявил discovery).

### 2. Чистота воронки - вес 15%

**Формула:** `(Сделки с заполненной «Дата решения клиента» И actualdate >= TODAY) / Сделки на «Принимают решение»`

Поле «Дата решения клиента» создаётся в Bitrix24 как обязательное при переводе на C49:3 (см. ТЗ Кости, робот «Фиксация даты»).

| Threshold | Verdict |
|---|---|
| ≥90% | Отлично |
| 70-89% | OK |
| 50-69% | Внимание |
| <50% | PIP-zone |

**Anti-cheat:** менеджер может заполнить «дата = сегодня + 365 дней» чтобы обмануть метрику. Кросс-чек: % сделок с датой решения >30 дней в будущем. Если >30% - алёрт.

### 3. Speed-to-lead - вес 10%

**Формула:** median часов между `Lead created` и первым касанием (исходящее письмо / звонок / комментарий менеджера).

| Median | Verdict |
|---|---|
| <1 час в рабочее время | Отлично |
| 1-3 часа | OK |
| 3-6 часов | Внимание |
| >6 часов | PIP-zone |

**Подсчёт:** только рабочее время (Пн-Пт 10:00-19:00 МСК). Лид, упавший в субботу в 23:00, должен быть touched до 11:00 понедельника.

### 4. Управление сроками - вес 10%

**Формула:** `Сделки с dwellCur > 14 дней без касания / Активные сделки`

`dwellCur` уже считается в `build-rop.ts:46-51`. «Без касания» = последний `touchReal` >14 дней назад.

| Ratio | Verdict |
|---|---|
| <10% | Отлично |
| 10-20% | OK |
| 20-35% | Внимание |
| >35% | PIP-zone |

**Calibration** [ГИПОТЕЗА: цифры estimate на основе текстов Костиных анализов, не извлечены прямой query rop.json. Требует пересчёта через `analytics-mvp/src/scripts/b24/build-rop.ts` на snapshot после 2026-06-30 cleanup]: у Масловой ratio ~49%, у Лысенко ~31%, у Виктории ~18% (GM). Median по отделу ~28%.

### 5. Алгоритм 3 касаний - вес 8%

**Формула:** `Сделки с просроченной эскалацией / Сделки требовавшие эскалацию`

Просроченная эскалация = робот «Эскалация» сработал и сделка простояла >3 дня без действия после.

| Ratio | Verdict |
|---|---|
| <5% | Отлично |
| 5-15% | OK |
| 15-30% | Внимание |
| >30% | PIP-zone |

### 6. Сделка-чистота (не свалка) - вес 7%

**Формула (эвристика):** сделка-свалка = `(distinct themes in comments + distinct assort fields used) >= 3 in last 60 days`.

`distinct themes` оцениваются по содержимому комментариев (NLP-keyword cluster, fallback на ручной spot-check 5 random сделок / квартал у каждого менеджера).

| Ratio свалок / активных сделок | Verdict |
|---|---|
| <5% | Отлично |
| 5-15% | OK |
| 15-25% | Внимание |
| >25% | PIP-zone |

**Calibration:** Маслова сделка 8 (229 дней, десятки запросов) - 100% классическая свалка. Алёна сделка 3 (100+ дней) тоже. Эти кейсы из Костиных отчётов - наш training set для эвристики.

### 7. Средний чек - вес 10%

**Формула:** median `budget` WON-сделок rolling 180 дней, по сегменту.

**Targets** [ГИПОТЕЗА: на основе плана `analytics-mvp/rop/plan/plan.json` (средний чек 80K глобально) + распределение по брендам из glossary.md; не валидировано на cohort WON-данных. Требует пересчёта после migration cleanup]:

| Cohort | Target | PIP-zone |
|---|---|---|
| B2B GENGLASS | 350K+ | <180K |
| B2C VALONTI | 120K+ | <60K |
| B2B GENTERO HoReCa | 800K+ | <400K |
| Дилер Metal-GM | 150K+ | <80K |

**Anti-cheat:** менеджер может специально не закрывать малые сделки чтобы держать средний чек высоким. Кросс-чек: количество WON в месяц. Если <5 - не считаем средний чек, считаем только close-win count.

### 8. Discipline дисконта - вес 5%

**Формула:** `% сделок WON с discount >5% за rolling 90 дней`

Discount = `(price_listed - price_won) / price_listed`. Price_listed из прайс-снимка соответствующего бренда.

| Ratio | Verdict |
|---|---|
| <10% | Отлично |
| 10-25% | OK |
| 25-40% | Внимание (разговор с Roman) |
| >40% | PIP-zone |

**Hard-rule:** дисконт >5% требует CFO sign-off (Roman). Это не зачётка - это проверка процесса.

### 9. Качество коммуникации - вес 10%

**Manual semi-quarterly review.** 5 random сделок WON + 5 random LOSE / квартал у каждого менеджера, оценка по 5-балльной шкале:

| Критерий | 1 | 3 | 5 |
|---|---|---|---|
| Приветствие + имя клиента | Никогда | Иногда | Всегда |
| Описание вложений | Пустые письма | Заголовки | Контекст + что внутри |
| Орфография | >2 ошибки/письмо | 1 ошибка | Чисто |
| Ответ на возражение | Шаблон / игнор | Один-два аргумента | Discovery question + аргумент |
| Эмпатия на жалобе | Восклицание / оборона | Извинение | Apology + что меняем |

**Калибровка scoring:** Юлия Мавлина сделка 5 («Синий цвет более тусклым!») = 1/5 эмпатия. Виктория сделка 5 (образцовая) = 5/5 по всем критериям. Эти кейсы - якоря шкалы.

| Score | Verdict |
|---|---|
| ≥4.5 | Отлично |
| 3.5-4.4 | OK |
| 2.5-3.4 | Внимание |
| <2.5 | PIP-zone |

## Сумма и пороги

**Weighted total** = Σ (score × weight). Каждый параметр на 0-10 шкале (5 brackets × 2 балла либо 1-5 × 2).

| Score | Цвет | Verdict | Действие |
|---|---|---|---|
| ≥9.0 | Зелёный | Top performer | SPIFF, рост ответственности, retention focus |
| 7.5-8.9 | Жёлто-зелёный | Solid | 1-on-1 coaching, target раз в 2 недели |
| 6.0-7.4 | Жёлтый | Внимание | Weekly 1-on-1, plan на улучшение 60 дней |
| 4.5-5.9 | Оранжевый | High risk | PIP design, 90 дней, exit-criteria |
| <4.5 | Красный | Не наш темп | Exit conversation в 14 дней |

## Cut-off для PIP (любой 1 из 4)

1. CR2 <50% от target cohort в течение 60 дней
2. Speed-to-lead >6 часов median 2 месяца подряд
3. >40% активного pipeline в zombie-state
4. Public quality incident (Костя-уровень анализа) не закрытый за 14 дней

## Anti-cheat (что менеджер может попытаться, как поймать)

| Cheat | Detection |
|---|---|
| Заполняет «дата решения» = сегодня + год | Алёрт на >30% сделок с датой >30 дней в будущем |
| Не закрывает мелкие сделки чтобы средний чек был высокий | Кросс-чек count WON / month |
| Перетягивает сделки коллег чтобы поднять CR2 | Audit `mgr` change history |
| Заполняет reason = «другое» массово | Алёрт на >50% «другое» в reason |
| Закрывает LOSE без даты и причины | Hard validation на C49:LOSE - reason обязателен |
| Многократно касается одного клиента в день чтобы поднять touchReal | Distinct days с касанием, не raw count |

## Reversibility

Если параметр поставлен ошибочно (например, я classified сделку как зомби, а она через 5 дней закрылась WON):

1. Корректировка metric в `knowledge/reflexion/YYYY-MM.md` с тегом SD-CORRECTION
2. Update эвристики если ошибка систематическая (>3 случая / квартал)
3. Public retract если менеджер был в PIP-zone по ошибке

## Anti-cheat: cross-cohort comparison guard

Hard rule: **не сравнивать менеджеров между сегментами**. Лысенко (B2B GENGLASS) vs Лакомова (B2C VALONTI) - запрещённое сравнение. Каждое сравнение должно быть либо within-cohort, либо нормализовано через cohort-median.
