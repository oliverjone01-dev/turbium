# Спецификации изображений Яндекс Директ

## API загрузки: `adimages.add`

Изображения передаются в base64 в поле `ImageData`.

## Допустимые размеры

### Стандартное изображение (обязательно для РСЯ)
| Формат | Размер (px) | Соотношение | Использование |
|--------|-------------|-------------|---------------|
| Квадрат | 1024×1024 | 1:1 | РСЯ, мобильные площадки |
| Квадрат (мин.) | 450×450 | 1:1 | Минимально допустимый |

### Широкоформатное изображение (рекомендуется)
| Формат | Размер (px) | Соотношение | Использование |
|--------|-------------|-------------|---------------|
| Wide | 1080×607 | ~16:9 | РСЯ десктоп, Telegram Ads |
| Wide (мин.) | 1080×607 | ~16:9 | Минимально для wide |

### Важно
- **Формат:** PNG или JPG
- **Макс. размер файла:** 10 МБ
- **НЕ принимается:** произвольный 16:9 (например 1376×768) — нужно ресайзить до 1080×607
- **Квадрат 1024×1024** принимается без проблем
- **Рекомендация:** загружать ОБА формата (square + wide) для максимального охвата площадок

## Привязка к объявлениям

```json
// Через ads.update
{
  "method": "update",
  "params": {
    "Ads": [{
      "Id": <ad_id>,
      "TextAd": {
        "AdImageHash": "<hash из adimages.add>"
      }
    }]
  }
}
```

## Рекомендации по содержанию

### ✅ Хорошо:
- Абстрактная визуализация (сети, узлы, геометрия)
- Чистый фон, минимализм
- Фирменные цвета
- Фото продукта/результата

### ❌ Плохо:
- Текст мелким шрифтом (не читается на маленьких площадках)
- Кликбейт (может не пройти модерацию)
- Изображения с водяными знаками
- Фото без прав (stock с watermark)

## Площадки РСЯ по форматам

| Площадка | Предпочтительный формат |
|----------|------------------------|
| Яндекс.Дзен | Wide 1080×607 |
| Telegram | Wide 1080×607 |
| Мобильные приложения | Square 1024×1024 |
| Десктоп сайты-партнёры | Wide или Square |
| Яндекс.Почта | Square 1024×1024 |
