# Humanizer-RU - 30 паттернов AI-следов в русском тексте

## When to invoke

- Любой русский текст ≥100 слов перед publish
- ОБЯЗАТЕЛЬНО для статей, landing copy, KP, email-кампаний
- Применять **двойным финальным проходом** (первый - структурный, второй - точечный)

## Принцип

Текст ≠ человеческий, если читатель за 2 минуты может сказать «это AI». Цель - убрать сигналы. Не упрощать, не «делать как блогер» - убирать **конкретные паттерны**.

> **Зачем это бизнесу (аргумент для Ивана).** В экспертный контент приходят за
> опытом и наработками автора; raw AI-текст читатель считывает и обесценивает. ИИ
> и ранжирует хуже контент, на котором «учиться нечему» (он сам такое генерит).
> Growth-практики вне нашей системы используют ИИ только для правки орфографии, а
> смысл пишут сами. Это оправдывает бюджет на humanizer-ru. [источник: К. Жаворонков]

## Block 1 - Канцелярит (8 паттернов)

| Паттерн | Замена |
|---|---|
| «в мире современного дизайна» | удалить, начать с сути |
| «не секрет, что» | удалить |
| «на протяжении веков / десятилетий» | удалить или конкретная дата |
| «является неотъемлемой частью» | переписать проще |
| «позволяет создать / реализовать / получить» | действие напрямую (создаёт, делает) |
| «представляет собой» | удалить, «это» или ничего |
| «в современных условиях» | удалить или конкретный год/контекст |
| «обладает свойствами» | конкретные свойства списком |

### Examples

❌ «GENGLASS представляет собой производителя, который позволяет создать уникальные интерьерные решения.»
✅ «GENGLASS делает зеркала и перегородки в Домодедово с 2018 года. 27 000 заказов.»

❌ «На протяжении веков стекло является неотъемлемой частью интерьеров.»
✅ «Стекло в интерьерах работает по двум функциям: пропускает свет и зонирует пространство.»

## Block 2 - Significance Inflation (6 паттернов)

| Паттерн | Замена |
|---|---|
| «уникальный» (без описания механики) | удалить или описать что именно уникально |
| «инновационный» | назвать технологию (например, «PVD-покрытие») |
| «революционный» | удалить, конкретика |
| «непревзойдённый» | удалить, бенчмарк |
| «эксклюзивный» | удалить или описать ограничение (например, «50 экземпляров») |
| «премиум» (без описания почему) | конкретика - материал, технология, гарантия |

### Examples

❌ «Уникальная коллекция, представляющая инновационный подход к дизайну.»
✅ «Линия TRUBIS использует вертикальные каннелированные основания - узнаваемый дизайн-код, который распознаётся без логотипа.»

## Block 3 - Promotional Fluff (8 паттернов)

| Паттерн | Замена |
|---|---|
| «высокое качество» | конкретные параметры (RAL, толщина, гарантия) |
| «опытные специалисты» | квалификация + число работ (например, «350+ установок») |
| «индивидуальный подход» | конкретное предложение |
| «доступные цены» | диапазон в рублях |
| «гармонично вписывается» | куда конкретно и почему |
| «идеальное решение» | для какой задачи и почему |
| «квинтэссенция» | удалить навсегда |
| «широкий ассортимент» | конкретное число моделей |

### Examples

❌ «Наши опытные специалисты предлагают индивидуальный подход и доступные цены.»
✅ «Конструктор GENGLASS Максим К. делает расчёт за 30 минут. Стандартная перегородка 2400×1000 - от 35 000 ₽, готова за 7 рабочих дней.»

## Block 4 - Структурные сигналы (5 паттернов)

1. **Em dash `-` запрещён.** Использовать дефис `-` или перестроить предложение.
2. **Предложения одинаковой длины.** Чередовать короткие (5–8) со средними (12–18) и длинными (20–25).
3. **Параллельные конструкции.** «Производство, доставка, монтаж» - лучше «Производим на 16 000 м². Везём по Москве за 1 день. Монтируем за смену.»
4. **Списки больше 7 пунктов.** Если >7 - разбить на 2 группы с подзаголовками.
5. **Абзацы >4 предложений.** Резать.

## Block 5 - Misc (3 паттерна)

1. **Двойные обороты вежливости.** «Уважаемый клиент, мы рады представить» - удалить оба.
2. **«Является» в любой форме.** Почти всегда лишнее.
3. **«Не только…, но и…»** - конструкция-маркер AI. Перестроить.

### Examples

❌ «Зеркало EVELIX не только украшает интерьер, но и решает функциональную задачу освещения.»
✅ «EVELIX делает две вещи: освещает (LED 3000K по периметру) и зонирует пространство (S/M/L размеры).»

## Workflow - Double Final Pass

### Pass 1 - Structural (5 минут)

1. Find & replace em dash → dash
2. Identify long paragraphs (>4 sentences) → split
3. Find lists >7 → reorganize
4. Check sentence length variety

### Pass 2 - Token-level (10 минут)

1. Grep canc patterns (block 1) → replace
2. Grep significance inflation (block 2) → replace
3. Grep promo fluff (block 3) → replace
4. Final read-through aloud (mentally) for naturalness

## Anti-patterns твоего выполнения

- ❌ Заменять одно AI-выражение на другое («уникальный» → «выдающийся» - это всё ещё inflation)
- ❌ Чрезмерное упрощение (терять конкретику)
- ❌ Игнорировать voice бренда (см. CLAUDE.md §6 МАРКО)
- ❌ Один проход (всегда двойной)

## Validation test

После прохода прогнать текст через 3 вопроса:
1. Default ChatGPT без промпта мог бы сгенерить такое же? (если да → REJECT)
2. Есть ли в каждом параграфе минимум одна конкретная цифра/деталь? (если нет → дописать)
3. Можно ли «вырезать» любой параграф и он останется самостоятельным smysl unit? (если нет → переписать)
