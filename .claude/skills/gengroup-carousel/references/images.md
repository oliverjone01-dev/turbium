# Картинки и графика в каруселях

Два типа визуала - два механизма. Не путать.

## 1. Данные и схемы -> CSS / inline SVG
Диаграммы, проценты, шаги, сравнения, числа - это слайд-типы `stat`, `process`,
`compare`, `list`, `checklist`. Делаются версткой, НЕ растром: резче на экспорте,
на бренд-токенах, ноль дрейфа между слайдами. Растровая «картинка-диаграмма» запрещена.

## 2. Реальная фотография и текстуры -> image slots (растр)
Фото изделий, интерьеров, материалов, цеха. Только это идёт картинкой.

## Как работает слот (фрейм с описанием ИЛИ файл - одно и то же)
Каждое фото живёт в `.imgslot` - фиксированная рамка. Раскладки: `.s-photo`
(full-bleed + подпись), `.s-inset` (полоса-фото над текстом), или слот внутри
любого слайда.

```html
<div class="imgslot">
  <img src="images/03.jpg" alt="" onerror="this.classList.add('missing')">
  <div class="ph">
    <span class="tag">фото · бриф</span>
    <div class="brief"><b>GENGLASS imagery:</b> ...описание кадра...</div>
    <span class="dim">images/03.jpg · 1080×1350</span>
  </div>
</div>
```

- **Файла нет** -> рендерится брендовый плейсхолдер + бриф (это и есть «фрейм с
  описанием»). Черновик экспортируется сразу, видно ТЗ на кадр.
- **Файл есть** -> слот сам показывает картинку, код не меняется.

## Два источника (один и тот же слот)
- **Свой аплоад:** сохранить в `images/NN.jpg` (или .png/.webp).
- **Генерация (Nano Banana 2 / Gemini Image / любой):** скопировать бриф слота ->
  сгенерить -> сохранить как `images/NN.jpg`.
- **Финальный / пересылаемый HTML:** вшивать как base64 data-URI (самодостаточно,
  переживает перенос файла). Для итераций относительный `images/` ок - экспортёр
  резолвит через file://.
- **Авто-генерация:** `scripts/generate_images.py carousel.html --style-lock "..."`
  парсит слоты, собирает промпт (бриф + style-lock) и вызывает Gemini/Nano Banana
  по env `GEMINI_API_KEY`, сохраняя файлы в `images/`. Без ключа или с `--dry-run`
  печатает готовые промпты под ручную генерацию. Модель по умолчанию
  `gemini-2.5-flash-image`; для Nano Banana 2 задать `GEMINI_IMAGE_MODEL=<id>`.
  `--force` перегенерирует существующие, иначе пропускает.

## Формат брифа (на каждый слот)
`[БРЕНД] imagery: <объект>, <ракурс>, <свет>, <обработка>, без людей. Style-lock: <замок>. <ШxВ>.`

## Style-lock (обязателен при генерации)
Один визуальный замок на всю карусель, применяется ко ВСЕМ брифам, чтобы кадры
не дрейфовали между слайдами: фиксируем свет + палитру + оптику + обработку.
Пример: «cool daylight, matte black RAL 9005, 50mm, no people». Именно это делает
генератор безопасным для премиума и снимает старую претензию «генераторы дрейфят».

## Imagery-DNA по брендам (из brand voice-DNA)
- **GENGLASS:** loft-industrial с премиальным финишем; муар на металле, отражения
  в стекле; cool daylight; без людей.
- **VALONTI:** музейная подача, драматичный свет, теневые провалы; материал крупно
  (Nero Marquina, Travertino Navona, Onyx Miele, бронза, латунь).
- **Metal-GM:** деталь крупным планом, цех, точность; сухой технический кадр; марки/RAL.
- **GLASS-MEMORY:** приглушённые тона, glass texture, без людей; деликатно, без
  скорбной атрибутики как продающего приёма.
- **GENTERO:** архитектура объекта, fit-out, схемы/чертежи; инженерная подача.

## Запрещено
- Сток «улыбающиеся люди», generic-градиенты, водяные знаки.
- Разный свет/палитра между слайдами одной карусели (нарушение style-lock).
- Данные растром - только CSS/SVG.

## Приёмка (ФЕНИКС)
Каждый заполненный слот: свет на бренде, людей нет (если бренд не разрешает),
текст читается поверх скрима, нет лишних тире, style-lock единый по всем слайдам.
