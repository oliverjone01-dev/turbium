Identify opportunities to add moments of joy, personality, and unexpected polish that transform functional interfaces into delightful experiences.

## MANDATORY PREPARATION

Use the frontend-design skill -- it contains design principles, anti-patterns, and the **Context Gathering Protocol**. Follow the protocol before proceeding -- if no design context exists yet, you MUST run teach-impeccable first.

## Delight Principles

### Delight Amplifies, Never Blocks
- Quick moments (< 1 second)
- Never delay core functionality
- Make delight skippable or subtle

### Surprise and Discovery
- Hide delightful details for users to discover
- Reward exploration and curiosity
- Let users share discoveries

### Appropriate to Context
- Match delight to emotional moment
- Respect the user's state
- Cultural sensitivity

## Delight Techniques

### Micro-interactions & Animation
- **Button delight**: Satisfying press effect, lift on hover
- **Loading delight**: Playful animations, personality in messages
- **Success animations**: Checkmark draw, confetti for achievements
- **Hover surprises**: Icons that animate, color shifts, tooltip reveals

### Personality in Copy
- Playful error messages (matched to brand)
- Encouraging empty states
- Product-specific loading messages (NOT generic AI filler like "Herding pixels")

### Illustrations & Visual Personality
- Custom illustrations for empty states, error states, loading
- Animated icons, illustrative icons
- Time-of-day themes, gradient mesh backgrounds

### Satisfying Interactions
- Drag and drop with lift effect and snap animation
- Toggle switches with smooth spring physics
- Form inputs that animate on focus
- Checkboxes with satisfying scale pulse

### Sound Design (when appropriate)
- Subtle notification sounds
- Satisfying success sounds
- Respect system sound settings, provide mute option

### Easter Eggs
- Konami code unlocks special theme
- Hover reveals on logos
- Console messages for developers
- Seasonal touches (holiday themes, weather-based variations)

### Celebration Moments
- Confetti for major milestones
- Animated checkmarks for completions
- Personalized messages ("You published your 10th article!")

**NEVER**:
- Delay core functionality for delight
- Force users through delightful moments
- Use delight to hide poor UX
- Overdo it (less is more)
- Ignore accessibility
- Sacrifice performance for delight
