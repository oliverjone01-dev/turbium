# Design Kit: единая точка входа с умным intake

Обёртка над остальными 23 скиллами Claude Design Kit. Сама код не генерирует:
сначала собирает контекст (intake), затем применяет один или несколько профильных
скиллов кита через инструмент Skill.

## Шаг 0. Intake (делать ПЕРВЫМ)
Перед любой маршрутизацией убедись, что известны 4 параметра. Если пользователь их
не назвал, задай ОДИН компактный вопрос батчем (не растягивай на диалог):

1. **Тип проекта**: лендинг, дашборд, SaaS, e-commerce, портфолио, админка, мобильное, иное.
2. **Индустрия/домен**: fintech, healthcare, beauty, e-com, образование и т.п. Влияет на
   палитру и индустриальные правила движка.
3. **Стек**: `html-tailwind` (дефолт), `react`, `nextjs`, `vue`, `svelte`, `shadcn`,
   `flutter`, `react-native`, `swiftui`, `jetpack-compose`.
4. **Эстетика**: minimal, brutalism, glassmorphism, premium-dark, playful и т.п.

Что не названо и не критично, бери по дефолту (стек по умолчанию `html-tailwind`) и
явно скажи, какой дефолт взял.

## Шаг 1. Дизайн-система (ui-ux-pro-max)
Всегда начинай отсюда. Запусти движок кита. Путь от корня репозитория именно
`.claude/skills/...`:

```bash
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "<тип> <индустрия> <эстетика>" --design-system -p "<Project>"
```

Для крупного или многостраничного проекта добавь `--persist` (паттерн Master + Overrides:
единый `design-system/MASTER.md` плюс overrides по страницам в `design-system/pages/`):

```bash
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "<query>" --design-system --persist -p "<Project>" [--page "<page>"]
```

Запуск Python упрётся в HITL-гейт (в `settings.json` `python3 <file>` не преаппрувлен).
Это ожидаемо, подтверди запуск.

## Шаг 2. Реализация (frontend-design)
Реализуй интерфейс в коде по выбранной дизайн-системе через скилл `frontend-design`,
стек берётся из intake.

## Шаг 3. Усиление (точечно, по необходимости)
Подключай профильные скиллы из таблицы маршрутизации ниже: цвет (`colorize`),
типографика (`typeset`), анимации (`animate`), раскладка (`arrange`), «вау» (`overdrive`),
радость (`delight`) и т.д.

## Шаг 4. Проверка перед релизом
`audit` (полный аудит качества: a11y, перформанс, темизация, адаптив), затем `polish`
(финальная доводка). Помни: китовый `audit` это дизайн-аудит интерфейса, он НЕ заменяет
адверсариальный гейт ФЕНИКСА (Step 12.5 / `feniks` / `phoenix-eval`).

## Полная таблица маршрутизации: задача → скилл

| Запрос | Скилл |
|---|---|
| Собрать дизайн-систему, выбрать стиль/палитру/шрифты/графики, «с чего начать» | `ui-ux-pro-max` |
| Реализовать выразительный интерфейс, компонент, страницу, артефакт | `frontend-design` |
| Полный аудит качества (a11y, перформанс, темизация, адаптив) | `audit` |
| Финальная доводка перед релизом (выравнивание, отступы, детали) | `polish` |
| Оценить дизайн с точки зрения UX (иерархия, ИА, резонанс) | `critique` |
| Слишком монохромно, добавить цвет | `colorize` |
| Типографика (выбор шрифтов, иерархия, читаемость) | `typeset` |
| Добавить анимации и микровзаимодействия | `animate` |
| Слабая сетка, ритм, иерархия раскладки | `arrange` |
| Дизайн скучный или безопасный, усилить | `bolder` |
| Слишком агрессивно или ярко, приглушить | `quieter` |
| Упростить, убрать лишнее | `distill` |
| Технически амбициозный «вау» (shaders, 60fps, физика) | `overdrive` |
| Моменты радости, характер, приятные мелочи | `delight` |
| Вынести переиспользуемые компоненты и токены в систему | `extract` |
| Привести к дизайн-системе, консистентность | `normalize` |
| Устойчивость: ошибки, i18n, overflow, edge cases | `harden` |
| Перформанс (загрузка, рендер, бандл, картинки) | `optimize` |
| Онбординг, пустые состояния, first-run | `onboard` |
| Адаптив под устройства и контексты | `adapt` |
| UX-копирайтинг, тексты ошибок, лейблы | `clarify` |
| Видео на React (Remotion) | `remotion-best-practices` |
| Один раз настроить дизайн-контекст проекта | `teach-impeccable` |

## Связка с GENGROUP (не отменяется)
Это инструмент дизайна, конституцию репозитория он не заменяет:
- Брендовый текст внутри интерфейса проводи через `brand` и `humanizer-ru`.
- Перед публикацией критического артефакта: Step 12.5, гейт `feniks`.
- Anti-Slop проверка применяется и к тексту в макете.
