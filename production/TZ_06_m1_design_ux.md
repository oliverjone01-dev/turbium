## 1. Задача (формулировка Ивана)

«Пройдись катком по дизайну, иллюстрациям, иконкам, баннерам - по десктоп и моб версии. Сделай понятную для людей концепцию, чтобы никто не ломал голову, но супер креативную и современную. Чтобы 60-летний собственник смог разобраться. Фишки и UX/UI - разработай ТЗ с правками вместе с ростером агентов и /design-kit».

Критерий приёмки: собственник 60 лет без маркетингового словаря понимает каждый экран за 5 секунд. Креативность не жертвуется - она канализируется в один понятный визуальный язык.

## 2. Визуальная концепция (КРЕА, принята ФЕНИКСОМ)

**«Приборная панель, которая показывает деньги как физические предметы».**

Два слоя:

1. **Приборный корпус** - то, что уже есть у m1 и работает: тёмный графит-зелёный, мятный акцент, срезанный угол Tb-65 (border-radius 10px 10px 10px 3px), консоль со счётом, моноширинные индексы. Это «панель прибора». Не трогаем, усиливаем.
2. **Физические метафоры денег** - каждая абстракция переводится в один осязаемый предмет:
   - Утечка заявок = **труба с монетами**, из которой на стыке выпадает монета (hero).
   - Хаос в заявках = чат и часы (заявка ждёт).
   - Слив бюджета = монета мимо кассы.
   - Зависимость от подрядчика = выдернутая вилка.
   - Непрозрачность = экран со знаком вопроса.
   - Честность = **стеклянная коробка** (всё видно).
   - Метод = **светофор** (зелёный - продолжаем, жёлтый - чиним, красный - останавливаемся).

**Правило одного предмета:** каждая иллюстрация - один физический объект, считываемый за 2 секунды, с подписью не мельче 13px. Если предмет нельзя назвать одним словом - иллюстрация переделывается.

**Anti-median тест:** дефолтная LLM-генерация даёт «абстрактные волны и градиенты». Труба с монетами, стеклянная коробка и светофор - нет. Пройдено.

## 3. Словарь ЭММЫ (канон текстов)

Единый словарь на все 11 страниц. Термин слева больше не употребляется без расшифровки:

| Было | Стало |
|---|---|
| лид, лидогенерация | заявка, привлечение заявок |
| пилот | пробный запуск (расшифровка при первом входе) |
| касание | обращение |
| контур | «у вас» |
| kill-критерий, стоп-сигнал | условие остановки |
| copilot | помощник |
| созвон | разговор |
| прототип (в контексте аудита) | аудит |
| GEO (голое) | видимость в поиске и AI-ответах (GEO) |

Ярлыки на цифрах - только два, единообразно: **[ГИПОТЕЗА]** для цен, **[ПРИМЕРНЫЙ РАСЧЁТ TURBIUM]** для расчётных чисел. Позиционирование canonical: «продуктовая команда» (не «агентство», не «реагент»).

## 4. Что реализовано в пакете m1-fix (11 файлов + assets)

### 4.1 Иконки и графика
- **Спрайт из 26 SVG-иконок** 24×24, штрих 1.75, инлайн в каждой странице (нет внешних запросов). Заменяют голые цифры и эмодзи-подобные маркеры в pain-strip, products, method, about.
- **Hero-сцена «труба заявок»**: 4 узла со срезанным углом (01 ЗАЯВКА → 02 ОТВЕТ → 03 ПОВТОР → 04 ПРОДАЖА), анимированные монеты едут по трубе, на узле 02 янтарная монета выпадает. Подпись: «Заявки едут по трубе. На шаге "Ответ" часть денег выпадает - это и есть утечка». `prefers-reduced-motion` отключает анимацию.
- **5 мини-сцен ситуаций** (Ха/Ра/Но/Ма/Бе) - SVG-виньетки 96×96 в строках ситуаций, скрываются ≤1120px чтобы не давить текст.
- **Светофор** на странице метода - визуальный якорь правила «продолжаем / чиним / останавливаемся».

### 4.2 Типографика и читаемость (порог «60 лет»)
- Ничего мельче 11px; основной текст на сером - не мельче 13px; на мобиле body 16px (не зумит iOS-форма).
- Консоль на hero: счёт «42 / из 100» получил расшифровку «Видно меньше половины пути ваших заявок» - цифра больше не загадка.
- Ярлык «ПРИМЕРНЫЙ РАСЧЁТ» прямо на консоли: демо-данные не выдаются за живые.

### 4.3 Навигация и структура
- Единая навигация на всех 11 страницах: 8 пунктов, включая Главную и Аудит; активная страница подсвечена (aria-current через data-page-link).
- **3 битые ссылки insights закрыты**: написаны статьи insight-leads.html, insight-crm.html, insight-agency.html - по схеме «одна проблема → один вывод → один шаг», без выдуманной статистики, с блоком «Честно про цифры».
- Пятая карточка ситуаций «Площадка забирает клиентов и комиссию» - закрыт разрыв между главной (4 ситуации) и solutions (5 решений).
- Цены: калькулятор потерь поднят ВЫШЕ прайса (сначала человек видит свою утечку, потом цену решения); заголовок «Три продукта. Цены открыты».

### 4.4 Мобильная версия
- Hero: порядок блоков - текст → труба → консоль (сцена-метафора раньше цифр).
- Консоль на мобиле схлопнута до 2 строк + кнопка «Показать весь путь» (аккордеон, JS 3 строки).
- Таблица ситуаций теряет арт-виньетки ≤1120px, сетка перестраивается.
- Тач-цели ≥44px, восстановлены asides системных шагов (grid-column:2, пунктирная граница).

### 4.5 Тексты
- Три текста resultCopy в site.js (итог квиза аудита) переписаны по-человечески - без «внедрение», «пилот», «управляемость», «GEO-контур».
- Все страницы прошли словарь ЭММЫ (§3) и humanizer-ru.
- Приватность квиза сохранена и проверена: расчёт полностью в браузере, ни fetch, ни storage не добавлены.
- Длинные тире удалены полностью (включая исходные диапазоны цен Ивана) - только дефис.

### 4.6 Санитария
- cache-bust `?v=7` на css/js.
- twitter:card метатег добавлен на главной.
- Футер: «Страница обновлена в июле 2026», «продуктовая команда · Москва / Россия».

## 5. Как залить (инструкция для Ивана)

1. Открыть репозиторий rayivan001-cell/m1 → корень.
2. Перетащить содержимое архива (11 .html + папка assets/) поверх существующих файлов. Структура зеркальная - GitHub предложит заменить.
3. Commit changes прямо в main. Pages пересоберётся за 1-2 минуты.
4. Проверить: главная (труба едет, счёт с подписью), pricing (калькулятор выше цен), любой insight (статьи открываются), мобильный вид главной (труба первой, «Показать весь путь» работает).

Папка assets/images/ в архиве не трогается - три jpg и favicon уже лежат в репозитории и остаются как есть.
